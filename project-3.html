Projet Universitaire

GÉNÉRATEUR D'AVATAR

SYMFONY BACKEND SSR

CONTEXTE DU PROJET

Commande Académique (BUT 3) : Développement d'une application "Clone de Gravatar" en quadrinôme.

Objectif : Créer un service centralisé de gestion d'avatars. L'application permet d'associer une photo à une adresse email et de la servir via une API publique sécurisée.

Thématique & Design : Réalisé durant le mois d'octobre, le projet arbore une charte graphique "Octobre Rose" (Camaïeu de roses, ruban symbolique) pour marquer notre soutien à la campagne de lutte contre le cancer du sein.

  • Équipe : 4 Collaborateurs (Met Gala Team)
  • Stack : Symfony, Twig, JS (Ajax natif), Sqlite

RÉALISATION & FEATURES

Le projet implémente un cahier des charges strict couvrant l'ensemble du cycle de vie des données utilisateur.

Sécurité & Privacy (BinaryFileResponse)

Les images ne sont pas stockées dans le dossier public. Elles sont servies via une route contrôleur /avatar/{hash} qui retourne une BinaryFileResponse. Cela permet de vérifier si l'utilisateur a masqué son profil avant de renvoyer l'image.

Expérience Utilisateur (AJAX)

Pour fluidifier la navigation, les vérifications d'unicité (email/login) et le basculement "Masquer profil" se font en JavaScript natif (AJAX), sans rechargement de page.

Administration & CLI

Intégration d'un mode maintenance (via variables d'env) et de commandes Symfony personnalisées pour la gestion des utilisateurs (ex: app:create-user) depuis le terminal.

BILAN DES COMPÉTENCES (NIVEAU 3)

COMPÉTENCE 1 : RÉALISER

SITUATION : Élaborer et faire évoluer une application complexe

AC31.02 - Conception & Architecture :

  • Problématique : Comment intégrer une logique métier complexe (hachage déterministe, fallback sur image par défaut, upload) sans polluer les contrôleurs ?
  • Solution : Implémentation d'une architecture Service-Oriented. Création d'un AvatarManagerService qui encapsule toute la logique de récupération/génération. Le contrôleur se contente d'injecter ce service et de retourner la réponse HTTP.
  • Preuve : Code source du contrôleur allégé (< 50 lignes) et Services testables unitairement.

COMPÉTENCE 2 : OPTIMISER (SPÉCIALITÉ)

SITUATION : Améliorer les performances en contexte contraint

AC32.03 - Bonnes pratiques & Bibliothèques :

  • Problématique : Servir des milliers d'avatars simultanément risque de saturer la bande passante et les I/O disque.
  • Solution : Mise en place d'un système de Cache HTTP avancé. Génération d'un ETag unique basé sur le hash du fichier. Si le client envoie un header If-None-Match correspondant, le serveur coupe court et renvoie une réponse 304 Not Modified (Body vide).
  • Impact : Réduction de 90% du trafic sortant pour les utilisateurs récurrents.

COMPÉTENCE 6 : COLLABORER

SITUATION : Organiser le développement en équipe agile

AC36.01 - Management de projet & Versionning :

  • Challenge : Travailler à 4 sur le même codebase Symfony entraîne des conflits (routes, entités).
  • Solution : Adoption du workflow Git Flow. Branches dédiées (feature/auth, feature/upload) mergées via Pull Requests avec Code Review obligatoire. Mise en place de conventions de nommage strictes pour les assets et les routes.
  • Résultat : Intégration continue fluide et zéro régression majeure en production.
