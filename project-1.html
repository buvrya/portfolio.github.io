CRITIQUES MET GALA

VUE.JS JAVASCRIPT IUT MONTPELLIER

CONTEXTE DU PROJET

Commande Académique (BUT 3) : Développement Full Stack d'une plateforme de critiques en Quadrinôme.

Sujet : "Met Gala Critiques". Une application permettant de noter et commenter les tenues de la célèbre cérémonie.

Architecture Headless : Séparation stricte entre le Back-end (API REST) et le Front-end (SPA). Le projet nécessitait une authentification robuste (JWT) et une intégration avec le service externe "MyAvatar".

  • Stack : Symfony 7, API Platform, Vue.js 3, Pinia.
  • Rôle : Lead Back-end & DevOps (CI/CD).

RÉALISATION & FEATURES

Une double application interconnectée offrant une expérience utilisateur riche et sécurisée.

API Platform & Sécurité

Développement d'une API REST de niveau 3. Gestion fine des droits avec les Voters (ex: Seul l'auteur peut modifier sa critique) et sécurisation via JWT & Refresh Tokens.

Client Vue.js Réactif

Interface SPA fluide consommant l'API. Utilisation de Pinia pour le state management (Utilisateur connecté, Favoris) et gestion globale des erreurs api.

Écosystème Connecté

Interconnexion avec le projet MyAvatar : hashage des emails utilisateurs (SHA-256) pour récupérer dynamiquement leurs photos de profil depuis notre service tiers.

BILAN DES COMPÉTENCES (NIVEAU 3)

COMPÉTENCE 1 : RÉALISER

SITUATION : Concevoir une architecture distribuée (Headless)

AC31.01 - Choix techniques & Intégration :

  • Problématique : Coupler un Front riche (Vue) et un Back robuste (Symfony) en gardant une indépendance totale.
  • Solution : Choix d'une architecture API-First avec API Platform. Exposition standardisée des ressources (JSON-LD/Hydra) permettant au front Vue.js de s'auto-configurer partiellement. Utilisation de StateProcessors personnalisés pour la logique métier complexe (ex: validation automatique des critiques admin).
  • Preuve : Documentation Swagger auto-générée et réutilisable par l'équipe Front.

COMPÉTENCE 2 : OPTIMISER (SPÉCIALITÉ)

SITUATION : Maîtriser l'exposition des données

AC32.02 - Qualité & Sécurité des données :

  • Problématique : Empêcher l'exposition de données sensibles (emails, hash mdp) via l'API publique.
  • Solution : Configuration fine des Normalization Groups. Définition de groupes contextuels (read:user:public vs read:user:owner) pour filtrer les champs sérialisés selon le rôle de l'appelant.
  • Impact : Sécurité robuste assurant qu'aucune donnée privée ne fuite dans les réponses JSON.

COMPÉTENCE 6 : COLLABORER

SITUATION : Coordonner une équipe Full Stack

AC36.02 - Co-conception & Intégration :

  • Challenge : Les développeurs Vue.js étaient bloqués en attendant les routes API finales.
  • Solution : Rédaction conjointe du Contrat d'Interface (OpenAPI) dès le jour 1. Mocking des routes critiques pour permettre au Front d'avancer sans attendre le Back.
  • Résultat : Développement parallèle efficace et réduction des "frictions" lors de la connexion finale Front/Back.
