ANALYSE EXPÉRIENCE

CHANTIER NAVAL COUACH

MONITORING AUTONOMIE TRAVAIL D'ÉQUIPE

INTRODUCTION & CONTEXTE

Stage au sein du service R&D du Chantier Naval Couach, autour du projet Magellan (Drone de surface / USV).
L'objectif était de développer une solution de monitoring et d'alerting embarquée pour surveiller l'état de santé du drone en temps réel (Capteurs, Moteurs, Serveurs).

Enjeu : Fiabilisation de l'infrastructure et réduction du temps de réaction en cas d'avarie technique lors des essais en mer.

RÉALISATION TECHNIQUE

Conception et implémentation de la chaîne complète de traitement de données (Data Pipeline).

Collecte (Python & SOLID)

Développement de scripts Python pour l'interrogation des capteurs (Modbus/Dataloggers).
Refactoring majeur : Passage d'un code scriptural à une Architecture Orientée Objet respectant les principes SOLID pour faciliter l'ajout de futurs capteurs.

Stockage (InfluxDB v2)

Mise en place d'une base de données Time Series (TSDB). Modélisation des données avec utilisation stratégique des Tags et Fields pour optimiser les performances des requêtes Flux.

Visualisation (Grafana)

Création de Dashboards interactifs pour l'équipe R&D. Configuration du système d'Alerting (Seuils critiques sur température/pression) avec notifications instantanées.

BILAN DE COMPÉTENCES (NIVEAU 3)

COMPÉTENCE 1 : RÉALISER

SITUATION : Refonte de l'architecture logicielle (Python)

AC11.01 - Qualité de Code & Design Patterns :

  • Problème : Script initial rigide et difficile à maintenir (Code duplicé pour chaque datalogger).
  • Solution : Application des principes SOLID (Responsabilité unique, Ouvert/Fermé). Création de classes génériques `Register` et d'interfaces pour les seuils (`Thresholds`).
  • Résultat : Code modulaire, testable et extensible sans modification du coeur logique.

COMPÉTENCE 2 : OPTIMISER

SITUATION : Performance du Monitoring Industriel

AC21.02 - Traitement de données temps réel :

  • Problème : Nécessité de traiter des flux de données continus sans latence pour l'alerting.
  • Solution : Optimisation des requêtes Flux sur InfluxDB v2 (Filtrage par tags indexés). Gestion de la persistance locale (Storage) pour conserver l'état des vues Grafana malgré le rafraîchissement automatique.
  • Résultat : Tableaux de bord fluides et alertes déclenchées en quasi temps réel sur anomalie.

BILAN & PERSPECTIVES

Ce stage a confirmé mon attrait pour l'Architecture Logicielle. La transition d'un code de script vers une architecture robuste (OOP/SOLID) a été une expérience formatrice majeure. J'ai pu mesurer l'importance de la qualité du code pour la pérennité d'un projet industriel.
